# Citation Grounding Policy

본 문서는 Clinical Research Harness가 출력하는 모든 인용에 대한 무결성 정책을 정의합니다.

## 절대 규칙

**모든 인용은 PMID 또는 DOI를 동반해야 한다.**
**LLM이 자유 생성한 인용은 출력 단계에서 자동 거절된다.**

## 근거

LLM의 의학 문헌 인용 환각률은 다수 연구에서 일관되게 30–50% 범위로 보고되었습니다:

- Bhattacharyya M, et al. "High Rates of Fabricated and Inaccurate References in ChatGPT-Generated Medical Content." Cureus 2023.
- Chelli M, et al. "Hallucination Rates and Reference Accuracy of ChatGPT and Bard for Systematic Reviews." JMIR 2024.
- Walters WH, Wilder EI. "Fabrication and errors in the bibliographic citations generated by ChatGPT." Sci Rep 2023;13:14045.

자유 생성 인용을 허용하는 순간 본 하네스 산출물 전체의 신뢰도가 무너지므로, 이 정책은 우회 불가능합니다.

## 허용된 인용 경로 (2가지)

### 경로 1: 도구 검증 인용
PubMed E-utilities, Semantic Scholar API, OpenAlex API가 직접 반환한 PMID/DOI.

### 경로 2: 사용자 명시 입력
사용자가 직접 PMID/DOI를 제공한 경우. 이 경우 출력 직전 PubMed/Crossref로 resolve 검증.

## 거절 대상 (예시)

다음 형태의 인용은 모두 거절됩니다:

```
❌ "Smith et al. (2024) reported similar findings."  (PMID/DOI 없음)
❌ "이 분야의 다수 연구가 보여주듯..."                (구체 인용 없음)
❌ "JACC 2023 논문에 따르면..."                      (구체 식별자 없음)
❌ "[Citation needed]"                              (placeholder)
❌ "추정 PMID: 99999999"                            (resolve 실패 시)
```

## 허용 형식

```
✅ Smith J, et al. JAMA 2024;331:1234-1241. [PMID: 38001234]
✅ Lee K, et al. Circulation 2023;147:e123-e145. [DOI: 10.1161/CIR.0000000000001234]
✅ (사용자 입력 인용, resolve 검증됨)
```

## 검증 단계

### 1. 생성 시점 (실시간)
literature-scout가 LLM 생성 텍스트에 PMID/DOI 패턴이 있으면 즉시 도구로 resolve.

### 2. 출력 직전 (post-hoc)
모든 출력 파일(`research_opportunities.md`, `feasibility_report.md`, `results.html` 등)을 스캔:
- 모든 [PMID: \d+] 패턴 → PubMed esummary로 존재 확인
- 모든 [DOI: ...] 패턴 → Crossref API로 resolve
- 실패 시 자동 거절 + evolution_log 기록

### 3. 사용자 검토 단계 (Phase 5 G5)
사용자가 결과를 검토할 때, 모든 인용에 PMID/DOI 링크가 동반되어 있어야 함.
링크 클릭으로 즉시 원문 확인 가능해야 함.

## 예외 처리

### Preprint (medRxiv, bioRxiv)
- DOI 동반 시 허용
- "preprint" 라벨 명시 필수
- 동료심사 미완료임을 명시

### 학회 abstract (ACC, AHA, ESC 등)
- DOI 또는 학회 abstract ID 동반 시 허용
- "conference abstract" 라벨 명시
- "full publication 미완료" 명시

### 비영어 문헌 (한국어 KMBASE 등)
- v1에서는 PubMed indexed만 지원
- 사용자가 직접 입력 시 KMBASE ID 또는 KCI 식별자 허용 (단, 자동 resolve 불가하므로 사용자 책임)

### 책, 가이드라인
- ISBN 또는 공식 URL 동반 시 허용
- 가이드라인 (예: ESC 2023): 공식 학회 URL 또는 PubMed 게재 시 PMID

## 환각 발견 시 처리

자동 탐지된 환각 인용은 다음 절차로 처리:

1. 출력에서 즉시 제거
2. evolution_log.md에 기록 (어느 Agent가 어떤 인용을 환각했는지)
3. 해당 Agent의 SKILL.md 가드레일 강화 후보로 표시
4. 사용자에게 "인용 X건이 검증 실패하여 거절되었습니다" 보고

## 한계 명시

- 본 정책은 인용의 **존재**를 보장합니다. 인용된 논문이 본문 주장을 **실제로 지지하는가**는 별도 검증 필요.
- 사용자 검토 단계에서 무작위 표본 인용을 직접 확인하는 것을 권장합니다.
- 일부 학술지는 PubMed에 등재되지 않거나 DOI가 없을 수 있음 — 이 경우 사용자 명시 입력 경로 사용.
